## Definition

**Recursion** is a pattern that calls itself.

An output becomes the next input, so the operation returns through the same rule across layers.

It is not just repeated words.

It is not endless mirroring.

Recursion builds structure by returning to itself, using what it produced to shape what comes next.

It can look like a question that contains its own method of answering, or a system evaluating itself and using that evaluation to update the next step.

Meaning stabilizes through recursion because the line does not merely repeat.

It returns through the same rule and becomes more structured each time.

**Recursion is self-return that uses what was produced to shape the next layer.**

## Canon Boundary

Under AI Foundations, recursion must not be mistaken for repetition, loop, mirroring, or circularity alone.

Repetition repeats.

Loop circles without resolution.

Mirror reflects.

Recursion returns through the same rule and uses the prior output to shape what comes next.

A recursive line becomes more structured through return.

If nothing is carried forward into the next layer, recursion has not occurred.
